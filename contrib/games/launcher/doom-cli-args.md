# Параметры запуска sdldoom (KolibriOS)

Этот список отражает реальные параметры, доступные в текущей сборке из этого репозитория.
Проверено по:
- `d_main.c`, `g_game.c`, `i_net.c`, `i_video.c`, `m_misc.c`, `p_spec.c`.

> Примечание: лаунчер пока запускает только Wolf3D и Quake; список Doom приведён для справки.

## Параметры (по блокам)

### Файлы и данные
- `@<file>` - response-файл с аргументами.
- `-file <wad1> [wad2 ...]` - добавить PWAD(ы) (до первого аргумента, что начинается с `-`).
- `-config <file>` - путь к конфигу (вместо `.doomrc`).
- `-cdrom` - использовать `c:\doomdata\` для сейвов/конфигов.

### Режимы игры / сложность
- `-skill <1..5>` - сложность.
- `-episode <n>` - стартовый эпизод.
- `-warp <map>` или `-warp <episode> <map>` - старт с указанного уровня.
- `-wart <episode> <map>` - совместимая форма `-warp`.
- `-turbo <percent>` - скорость игрока в процентах.
- `-nomonsters` - без монстров.
- `-respawn` - монстры возрождаются.
- `-fast` - режим fast-monsters.
- `-deathmatch` - deathmatch.
- `-altdeath` - alt-death (без выхода из игры).
- `-timer <minutes>` - лимит времени (для deathmatch).
- `-avg` - вывод средних значений (для deathmatch).

### Демо, тесты и производительность
- `-record <name>` - запись демо.
- `-playdemo <name>` - воспроизведение демо.
- `-timedemo <name>` - timedemo (тест скорости).
- `-loadgame <slot>` - загрузить сохранение (номер слота).
- `-maxdemo <kB>` - максимальный размер демо (в KB).
- `-statcopy <addr>` - копировать статистику во внешний буфер.
- `-nodraw` - выключить рендер.
- `-noblit` - выключить blit.
- `-debugfile` - писать debug-файл `debug<player>.txt`.
- `-devparm` - developer-режим.
- `-shdev`, `-regdev`, `-comdev` - developer-наборы данных.

### Сеть
- `-net <player> ...` - netgame-параметры.
- `-port <p>` - сетевой порт.
- `-dup <n>` - ticdup.
- `-extratic` - extra tic для синхронизации.

### Видео и ввод
- `-fullscreen` - полноэкранный SDL-режим.
- `-2` - масштаб 2x от 320x200 (640x400). Значение по умолчанию.
- `-3` - масштаб 3x от 320x200 (960x600).
- Режим 1x (320x200) недоступен: `multiply` по умолчанию равен 2, флага `-1` нет.
- `-grabmouse` - захват мыши.
